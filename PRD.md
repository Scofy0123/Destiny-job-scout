## 一、系统愿景与使命 (Vision & Mission)
### 1.1 问题陈述
1. **信息获取效率瓶颈**：高端求职者需要在海量包含“日结、外包、实习”的冗余数据中耗费极大精力，才能提炼出有效的高薪岗位线索。
2. **缺乏“直击心灵”的契合度判决**：普通的自动化爬虫仅仅搬运信息，并不“懂”用户。它无法结合用户的职业内核给出具有深度共鸣、个性化的推荐排雷建议。

### 1.2 Mission 宣言
**Boss Job Scout** 旨在成为高阶求职者的全自动私人 AI 猎头战车。系统通过仿生学雷达爬虫无缝穿透风控，利用大模型结合重度个人画像，实施“千人千面”的毒辣审判，最后向求职者的移动端推送“一剑封喉”的岗位简报。

### 1.3 成功定义
1. **基础自动化**：全程 0 干预运作（Cron/后台静默）。
2. **反爬存活率**：深潜阶段 100% 绕过/免疫风控滑块。
3. **推片转化率**：大模型生成的私语点评直接刺中求职者痛点，降低无效面试率 80% 以上。

## 二、系统画像与信任边界
### 2.1 Agent 定位
本系统属于典型的 **Level 3 (自主型助手)**：无需频繁提问，通过 `_default.json` 获取顶层指令后，能够独立执行复杂的跨系统编排（爬虫搜寻 -> 深度钻取 -> AI 研判 -> 飞书推送）。

### 2.2 信任与自治水平
- **只读级代理权**：拥有代用户爬取公开网络和调用本地 LLM 的权力。
- **限制级操作权**：仅被授予对私人飞书（Bitable、IM）的写入权限。系统不具备代办投递等写保护行为的能力。

### 2.3 关键利害关系人
- 用户（求职者本人）；Boss 直聘风控引擎（对抗方）；大模型推理引擎（LLM Judge）。

## 三、模型与认知架构
### 3.1 预训练模型选型
放弃调用容易导致隐私阻断和能力受限的开源 API，**本系统直接依托于全栈沉浸式 Agent 的原生大语言模型 (Agent-Native LLM)**。这赋予了系统极限的上下文连贯性和最高的交互智商，实现与求职者最原生的深度共鸣。

### 3.2 Context 管理与路由策略
系统采取严格的分层认知拦截机制：
1. `resume.txt`（客观硬技能：履历、项目经历）
2. `_my_profile.md`（主观软体质：职场野心、雷区爆点、降维杠杆）
3. `topic_results_detailed.json`（经过两次风控筛选后的原始近千字 JD）

**Prompt 重组策略**：
将上述三域数据进行“浓缩封测”，压入单一回合系统 Prompt。强制采用 “Role: Assistant” 日志切割法，过滤掉冗杂的内部思维链，只向外滩输出 “合伙人私语”。

## 四、工具与行动空间 (Tools)
这是一个典型的 **Tool-centric Architecture (以工具为中心的架构)**：

### 4.1 工具清单

| 能力组件 | 提供方/底层机制 | 功能契约 |
| :--- | :--- | :--- |
| **浅层全域扫盲** | `opencli boss search` | 给定城市、门槛，输出列表 JSON 级浅层毛数据。|
| **深海抗压萃取** | `opencli boss detail` | 传入 `security_id`，穿透获取无码完整版 JD 文本。 |
| **LLM 研判器**   | `Agent-Native LLM` | 对抗型分析器，寻找契合点与毒辣劝退点。 |
| **归档枢纽**     | `lark-cli bitable`/`im` | 提供不可变的结构化存储与移动端触达通道。 |

### 4.2 反风控执行单元方案
为应对高强度爬虫拦截，深海工具引入 **4 层反风控套件**：
1. 宽带抖动：`25~45` 秒随机休眠波浪。
2. 咖啡局（Chunking Break）：每连续抓取 5 次强制假死 `60~120` 秒。
3. 动态配置封箱：不再依赖硬编码爬虫限制。
4. 无头 CLI 绝缘：底层直连接口，隔离鼠标监控探针。

## 五、环境与编排架构
系统的工作流拆解为四个顺序编排的工业级管线。

### 5.1 编排策略
采用 **Pipeline 线性图灵机 (DAG)** 模式。
- **Phase 1 (Basic Radar)**：拉网式搜索 → 硬性正则筛查（含实习/门槛拦截） → 存入表格。
- **Phase 2 (Deep Scrape)**：限流下探（根据 `max_deep_scrape_limit`） → 滑动窗口防封抓取。
- **Phase 3 (Brain Judge)**：将下探战利品投喂至大模型并挂载 `DNA`。
- **Phase 4 (Lark Push)**：正则切片 → 高级飞书卡片送达。

## 六、评估与验证
1. **防风控存活率 (Survival Rate)**：连续抽取 20 单长篇 JD 的成功率应 > 95%。
2. **命中有效率 (Hit Rate)**：剔除日结/不符预期的比例应维持在 100%（由 Python `re` 引擎底盘兜底）。
